# Vault Features - Shifu

This file writes specifications for changes to the vault features of the Shifu application.

**Date:** 2026-07-17
**Summary:** The vault currently only holds flashcard questions for learning, and does not allow storage of an information database for work. The new vault should determine group all information of windows and data into specific projects/tasks the user is working on and write detailed information logs of what occurs and is accomplished by the user during sessions.


### Main Interface ###

The vault interface should include a section that shows compiled and easy to consume logs/summaries of today’s work so far. The interface should also show what the most recent tasks performed/accessed were (and a very brief explanation).

Spaced repetition can be a separate menu screen, which allows greater customization features of what data/tasks to pull from as decks.

### Tasks ###
Group the users actions into tasks and log all information over time into these tasks. Tasks can span multiple days and extended periods of time, and the user should be able to rename these tasks.

### Projects ###
The user should be able to create **projects**, which will be a major feature of the application. Projects group tasks together into direct goals that the user is trying to achieve (for example a learning goal, work goal, and other extended efforts).

The user should be able to see time spent on these projects and have spaced repetition or separate information “vaults” that store project specific information and status.
